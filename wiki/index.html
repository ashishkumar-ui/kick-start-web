Kick Start Web is a light weight UI framework packed with lot of features integrated in it. This is formed for very custom needs to start a RESTful web application. This framework is an integrated suite of all the good web-tools avaialble.

Here are the features listed of it:

  • Modular code
  • Uses RequireJS
  • Ajax service wrapper
  • Templating Manager
  • REST API standalone configuration
  • Application level standalone configuration
  • Internationlizaton (i18n) integrated for base HTML and templating
  • Shared Scope cross modules for sharing data, functions etc.
  • Common utility available

Modular code

The framework contains clean architecture of very modular code. It contains a /js folder with below contents in it:

  1. /globals: All the JS files which are being used globally like variuos global configurations, template manager, translation manager, service layer wrapper etc.
  2. /modules: Keeps all the module/component specific JS files in it.
  3. /vendors: Keeps all the 3rd party JS files.
  4. app.js
  5. require-config.js: global requirejs configurations

Idea is to keep each file module/component specific and each of them is supposed to expose only required functionalities outside with the help of requirejs

Uses RequireJS

As mentioned above, it uses RequireJS to bind up the entire architecture, which helps to make the entire app modular easily.

Ajax service wrapper

It has an Ajax wrapper module which helps to have the REST service call with some sort of central approach. You can set variuos default options for all the Ajax calls like error/success handler, pre-ajax setups etc. at one place.

All you need to inject the service module with RequireJS like:

var http = require("http");

And then use the exposed API methods based on your needs:

http.get(options);
http.post(options);

Templating Manager

With the help of a very popular and commonly used templatig engine known as handlebar, we have built the templating manager which parses all the templates in the page and stores in a some sort of repository and then you can use the same whenever you need any of theose. You also can define partials and use them within the templates too.

REST API standalone configuration

The framework has a central configuration for REST URLs where you can keep all your service URLS at one place and then you can use it wherever you want. All you need to do is to inject this configuration file in your module where you need it var serviceConfig = require("serviceConfig");

And then all the configured URLS are available to this module which can be used further:

           var http = require("http"), // Injecting Service API
               serviceConfig = require("serviceConfig"), // Injecting Service Configuration
               ajaxParams;
               
            ajaxParams = {
                url: serviceConfig.module1.getTaskList, // use service URL
                data: {
                    pid: 20
                },
                successCallback: function(resp){
                    // do something here
                }
            };
            
            // Make Service call
            http.get(ajaxParams);
